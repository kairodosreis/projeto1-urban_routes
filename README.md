# 🧪 Projeto 1 - Casos de Teste e Relatórios de Bug
Projeto acadêmico de criação de casos de teste e documentação de bugs para o aplicativo Urban Routes.

---
 
## 🚀 Começando  
Este repositório contém os casos de teste e relatórios de bug realizados no aplicativo Urban Routes. O objetivo foi validar funcionalidades de navegação, pesquisa de endereços e modos de visualização do mapa.  

---

## ⚙️ Executando os testes  
Os testes foram realizados manualmente e documentados em checklist.  

### 🔩 Testes de ponta a ponta  
Verificam o fluxo completo do usuário:  
- Rolagem e zoom do mapa  
- Pesquisa de endereços nos campos "De" e "Para"  
- Exibição de objetos 3D, parques, pontos de referência e estações de metrô  
- Ativação de modos de visualização (Satélite, Relevo, Street View, Tela Cheia)  

Exemplo:  
- **CASO-3 (URB-1)**: títulos das áreas no mapa não deveriam ser clicáveis, mas estavam.  
- **CASO-7 (URB-3)**: pino de endereço não aparece após preenchimento do campo "De".  

### ⌨️ Testes de estilo de codificação  
Foram aplicados padrões de consistência na interface:  
- Botões de modo de visualização ativando/desativando corretamente  
- Ícones exibidos conforme o design ao aplicar zoom  
- Informações do aplicativo exibidas ao clicar no logotipo (bug URB-8)
